## 口說同步機制

每個視覺段落有**獨立的 TTS 音頻檔**：
- `seg_pain.mp3` → 痛點畫面
- `seg_win.mp3`  → 成果畫面
- `seg_step1_type.mp3` → Step1 打字畫面
- `seg_step1_out.mp3`  → Step1 輸出畫面
- `seg_stepN_type.mp3` / `seg_stepN_out.mp3` × 3
- `seg_cta.mp3`  → CTA

視覺片段長度 = 音頻實際長度 → **永遠精準同步**

聲音：Microsoft Edge TTS `zh-TW-HsiaoChenNeural`（台灣女聲）

## 內容規則

### ✅ 必須做到
- 每步驟包含「**真實可貼用的 Prompt 範本**」（4-5行，含背景＋需求＋格式＋語氣）
- 每步驟包含「**AI 實際輸出範例**」（6-8行，有具體數字/日期/格式）
- 情境貼近台灣職場：Email、會議記錄、簡報、報表、客戶溝通
- 旁白字數：痛點≤35字、成果≤35字、每步打字段≤25字、每步輸出段≤25字、CTA≤30字

### ❌ 絕對禁止
- 出現任何中文真實人名（用「主管」「客戶」「同事」「業務」代替）
- 空洞行銷語言（「超強」「改變人生」「神器」「非常方便」）
- Prompt 範本是說明文字而非可用指令
- 口說內容與畫面不一致

## 技術架構

```
Gemini 2.5 Flash → 生成腳本（含分段旁白）
Edge TTS         → 每段生成獨立 mp3
video_renderer   → 動態打字動畫 + 輸出串流
moviepy          → 拼接 + 輸出 mp4
```

## 執行模式

- **FULL**（手動觸發）：確認無誤後上傳 YouTube
